Zeno UI

A beautiful, responsive UI library built with vanilla CSS, HTML, and JavaScript. Works great on mobile and desktop.

Philosophy

Why Zeno UI exists and the principles that guide its design.

In a world of complex build tools, heavy frameworks, and endless configuration, Zeno UI exists to bring back simplicity. We believe that building beautiful interfaces shouldn't require learning a new framework, setting up a build pipeline, or wrestling with dependencies. Sometimes, you just need components that work—right now, right out of the box.

Simplicity Over Complexity

No build tools. No dependencies. No configuration files. Just pure HTML, CSS, and JavaScript that you can copy, paste, and customize. We've made the hard decisions so you don't have to.

Opinionated by Design

We provide sensible defaults—from our warm earth-tone color palette to our minimal border radius choices. Our layout system eliminates unnecessary wrappers (no row divs needed!). These opinions aren't arbitrary; they're the result of careful consideration about what works best in practice.

Mobile-First, Always

Every component is built with mobile in mind first, then enhanced for larger screens. Responsive design isn't an afterthought—it's woven into the fabric of every component, ensuring your interfaces work beautifully on any device.

Progressive Enhancement

Zeno UI embraces the web platform as it is. We use modern CSS features like custom properties and the `:has()` selector, but we don't require JavaScript for basic functionality. Components degrade gracefully and enhance progressively.

Design System Thinking

Every spacing value, color, and transition is defined as a CSS custom property. This creates a cohesive design system that's easy to customize and maintain. Change a variable once, and it updates everywhere—the way design systems should work.

Zeno UI is for developers who value their time, appreciate thoughtful defaults, and want to build great interfaces without the overhead. It's not trying to be everything to everyone—it's trying to be exactly what you need, when you need it.

Screenshot Component

Capture elements as images using native browser APIs - no dependencies required!

Important: This component uses native browser APIs which have security restrictions. SVG foreignObject (required for capturing styled HTML) is blocked by many browsers. For production use with complex styling, consider using a library like html2canvas. This component works best for simple elements or as a fallback.

Capture This Card

This is a demo card that you can capture as a screenshot. The screenshot component uses pure browser APIs (Canvas, SVG) with zero dependencies.

Capture Entire Page

Usage

<!-- Capture specific element -->
<button data-screenshot="#my-element" data-screenshot-filename="my-screenshot">
  Capture Element
</button>

<!-- Capture full page (no selector) -->
<button data-screenshot data-screenshot-format="png">
  Capture Page
</button>

<!-- Programmatic API -->
// Capture element
const blob = await ZenoScreenshot.captureElement('#my-element', {
  format: 'png',
  quality: 1.0,
  backgroundColor: '#ffffff'
});

// Copy to clipboard
await ZenoScreenshot.copyToClipboard(document.getElementById('my-element'));

Options

  • data-screenshot - Element selector (or empty for full page)
  • data-screenshot-format - Image format: "png" (default) or "jpeg"
  • data-screenshot-quality - Quality (0-1) for JPEG format
  • data-screenshot-filename - Download filename (default: "screenshot")
  • data-screenshot-bg - Background color (default: "#ffffff")
  • data-screenshot-download - Set to "false" to prevent auto-download

Typography Class Reference:

  • .title - Main page title (responsive: 1.5rem → 2.25rem)

Content Class

The .content class provides beautiful styling for semantic HTML content like articles, blog posts, and documentation. It automatically styles headings, paragraphs, lists, code blocks, tables, and more.

Beautiful Content Styling

The .content class automatically styles all semantic HTML elements for optimal readability. It sets a comfortable max-width of 65 characters per line and provides consistent spacing throughout.

Headings Hierarchy

All heading levels are properly styled with appropriate sizes and spacing. The class ensures a clear visual hierarchy that guides readers through your content.
